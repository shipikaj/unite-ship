# Alerts

The **Alerts** section helps you stay informed about changes and events occurring across your Unite workspace.

Alerts provide visibility into pipeline activity, integration changes, subscription updates, and system events, helping teams identify issues quickly and maintain reliable data operations.

---

# Why Alerts Matter

In a production ETL environment, unexpected changes can impact reporting, dashboards, and downstream systems.

Alerts help you:

- Monitor pipeline activity
- Track source and destination changes
- Stay informed about subscription events
- Identify operational issues early
- Reduce troubleshooting time

We recommend enabling alerts for all users responsible for managing data pipelines and integrations.

---

# Configure Alert Preferences

Notification settings can be customized based on the types of events you want to monitor.

Each user can maintain their own notification preferences, ensuring they receive only the updates relevant to their responsibilities.

---

# Recommended Alert Categories

## Data Sources

Receive notifications when source integrations are added, removed, or updated.

Recommended for:
- Data engineers
- Workspace administrators
- Analytics teams

Typical events include:

- New data source connected
- Data source removed
- Integration updates

---

## Destinations

Receive notifications when destination systems are modified or encounter connection issues.

Recommended for:
- Data engineers
- Reporting teams
- Platform administrators

Typical events include:

- New destination configured
- Destination removed
- Connection failures
- Authentication issues

---

## Connections & Pipelines

Receive notifications related to synchronization workflows.

Recommended for:
- Data teams
- Analytics engineers
- Platform operators

Typical events include:

- Pipeline created
- Pipeline deleted
- Pipeline paused
- Pipeline resumed
- Synchronization failures
- Synchronization completed

---

## Plans & Subscription

Receive updates related to billing, plan changes, and workspace usage.

Recommended for:
- Workspace owners
- Billing administrators

Typical events include:

- Trial expiration reminders
- Subscription renewals
- Plan upgrades
- Usage limit notifications

---

## System Alerts

Receive notifications about platform-wide operational events.

Recommended for:
- Workspace administrators
- Technical owners

Typical events include:

- User/Admin addition or deletion
- System Outage / Downtime
- Performance Degradation Warning
- Feature Update / Deprecation Notice
- User Role Updated

---

# Review Notification History

The **Notifications** tab provides a centralized activity feed containing recent workspace events.

Use this section to:

- Review pipeline activity
- Confirm integration updates
- Track operational changes
- Investigate recent actions

The activity feed serves as an audit trail for important workspace events.

---

# Best Practices

For most teams, we recommend enabling notifications for:

- Pipeline activity
- Source integration changes
- Destination connection issues
- Subscription and usage alerts

These notifications provide early visibility into issues that may affect reporting and data availability.
