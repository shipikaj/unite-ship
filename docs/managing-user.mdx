# User Management

The **User Management** section allows workspace administrators to control who can access Unite and what actions they can perform within the workspace.

Use this section to invite team members, assign roles, review user access, and maintain secure collaboration across your data operations.

---

# Why User Management Matters

Different team members require different levels of access depending on their responsibilities.

User Management helps ensure that:

- Users have access to the tools they need
- Sensitive settings remain protected
- Workspace operations remain secure
- Administrative responsibilities are properly controlled

This allows teams to collaborate effectively while maintaining governance over pipelines, integrations, and workspace settings.

---

# Invite New Users

To add a user to your workspace:

1. Click **Add New Users**
2. Enter the user's email address
3. Select a role
4. Send the invitation

The invited user will receive an email with instructions to join the workspace.

Users can be added directly through the side drawer invitation flow without leaving the page.

---

# User Roles

Every user is assigned a role that determines the level of access available throughout Unite.

| Role | Typical Responsibility |
|--------|-----------------------|
| Super Admin | Workspace ownership and full administrative control |
| Admin | Workspace administration and operational management |
| Developer | Technical management of pipelines and integrations |
| Analyst | Monitoring and analysis with limited operational actions |

The capabilities available throughout Unite depend on the assigned role.

---

# Manage Existing Users

The user table provides a centralized view of all workspace members.

For each user, administrators can view:

- Name
- Email address
- Assigned role
- Invitation status

This allows administrators to quickly review and manage workspace access from a single location.

---

# Update User Roles

As responsibilities change, administrators can update a user's role.

Role changes take effect immediately and impact access across:

- Data Sources
- Destinations
- Pipelines
- Alerts
- Account Settings

Administrators should periodically review assigned roles to ensure users have the appropriate level of access.

---

# Pending Invitations

Users who have not yet joined the workspace appear as pending invitations.

Administrators can resend invitations when necessary.

Common reasons include:

- Invitation email was not received
- Invitation link expired
- User accidentally deleted the invitation email

---

# Remove Users

Users can be removed when workspace access is no longer required.

Removing a user immediately revokes access to:

- Data Sources
- Destinations
- Pipelines
- Alerts
- Workspace Settings

Historical activity remains available within workspace records and logs.

---

# Access Management Best Practices

To maintain a secure workspace, we recommend:

- Assigning the lowest level of access required
- Reviewing active users regularly
- Removing inactive users promptly
- Limiting administrative access to trusted personnel
- Reviewing pending invitations periodically

Following these practices helps maintain operational security while enabling effective collaboration.
