# Connect Amazon Redshift

This guide explains how to configure Amazon Redshift Serverless as a destination in Unite.

Once connected, Unite can create tables and synchronize data from your pipelines into your Redshift data warehouse.

---

## Before you begin

Ensure you have the following:

- An Amazon Redshift Serverless database.
- Access to the AWS Management Console.
- Permission to manage Redshift Serverless resources and security groups.
- A Unite user account with permission to create and manage destinations.

---

## Connect your destination

### Step 1 - Retrieve your connection details

From the Amazon Redshift console, locate the following information:

- Endpoint (Host)
- Port
- Database Name
- Admin Username

> **Note**
>
> The endpoint may include the host, port, and database name in a single value. Enter each value separately when configuring the destination in Unite.

---

### Step 2 - Allow Unite to connect

To allow Unite to connect to your Redshift instance:

1. Open your Redshift **Workgroup**.
2. Navigate to **Network and Security**.
3. Open the associated **VPC Security Group**.
4. Edit the **Inbound Rules**.
5. Add a **Custom TCP** rule for your Redshift port.
6. Allow inbound access from the Unite IP addresses for your region.

This enables Unite to establish a secure connection to your Redshift warehouse.

---

### Step 3 - Choose a database user

You can connect using either:

- **Master User** (recommended)
- **Limited User**

#### Option A — Master User

Use your existing Redshift administrator credentials.

You'll need:

- Host
- Port
- Database Name
- Username
- Password

---

#### Option B - Limited User

If you prefer not to use the administrator account, create a dedicated user for Unite.

Create the user:

```sql
CREATE USER unite PASSWORD '<password>';
```

Grant database permissions:

```sql
GRANT CREATE, TEMPORARY
ON DATABASE "database"
TO unite;
```

Grant schema permissions:

```sql
GRANT USAGE, CREATE
ON SCHEMA "schema"
TO unite;
```

Grant table permissions:

```sql
GRANT SELECT, DROP
ON ALL TABLES IN SCHEMA "schema"
TO unite;
```

Replace:

- `database` with your database name.
- `schema` with your schema name.
- `<password>` with a secure password.

---

### Step 4 - Configure the destination in Unite

1. Open **Destinations**.
2. Click **Add Destination**.
3. Enter a destination name.
4. Select **Amazon Redshift**.
5. Enter the following information:

- Host
- Port
- Database Name
- Username
- Password
- Schema
- Time Zone

6. Click **Save & Test**.

---

## Verify the connection

Unite validates the destination by checking:

- Network connectivity
- Authentication
- Database accessibility
- Schema permissions
- Table creation permissions

If validation succeeds, the destination status changes to **Complete** and is ready to be used in pipelines.

---

## Next step

After successfully connecting Amazon Redshift, create a pipeline to begin syncing data from your connected data sources.
