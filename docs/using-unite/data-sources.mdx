# Data Sources

The **Data Sources** section is used to manage platforms where data is extracted from within Unite.

This section provides visibility into connected accounts, integration health, source status, and connection activity.

---

# Overview

Data Sources are the starting point for all synchronization workflows in Unite.

Before a pipeline can be created, at least one source integration must be connected and authorized.

Supported source categories include:
- Analytics platforms
- Advertising platforms
- Ecommerce platforms
- CRM systems
- Internal business systems

Supported integrations include:
- Google Analytics 4
- Meta Ads
- Google Ads
- Google Seach Console
- Shopify
- HubSpot

---

# Source List

The Data Sources page displays all connected source accounts in a centralized table view.

Each source includes:
- Connected account
- Integration type
- Connection status
- Creation timestamp

This allows teams to review workspace integrations and monitor operational readiness from a single interface.

---

# Source Status

Each source displays a status indicator representing its current connection state.

### Complete

The source has been connected successfully and is ready for pipeline creation and synchronization.

### Incomplete

The source setup requires additional configuration or authorization before it can be used.

### Error

The source connection encountered an authentication or operational issue that requires attention.

---

# Connect Data Source

Users can connect a new integration directly from the **Connect Data Source** action available on the page.

The connection process includes:
- Integration selection
- Account authentication
- Permission approval
- Workspace authorization

Available integrations depend on the active workspace plan.

---

# Search and Filtering

The Data Sources section includes search and filtering tools for managing connected accounts efficiently.

Users can:
- Search connected accounts
- Filter by integration type
- Review connection status
- Navigate integrations quickly

---

# Source Management

Users can manage integrations directly from the table view.

Available actions include:
- Reconnecting expired accounts
- Removing integrations
- Reviewing connection details
- Monitoring integration health

Changes to source integrations affect associated pipelines connected to those accounts.

---

# Operational Notes

- A source must be connected before creating pipelines
- Authentication failures interrupt synchronization workflows
- Removing a source affects associated pipelines
- Some integrations require periodic reauthorization
